Welcome!

Welcome to Jakarta EE! This is a very simple Jakarta EE application generated by the official Eclipse Starter. It is a simple RESTful CRUD service that uses a representative set of Jakarta EE technologies such as REST, Enterprise Beans, CDI, Persistence, JSON Binding and Bean Validation. The application is a cafe that has an inventory of coffee. You can easily consume the RESTful service from a popular JavaScript framework such Angular, React or Vue. The content type used by the service is JSON.

The REST end-point is available at rest/coffees. The end-point allows you to:

  • List all the current coffee varieties by issuing a GET request to rest/coffees. This is how the output will look like: 
    [
    {
        "id":1,
        "name":"Coffee-A",
        "price":2.75
    },
    {
        "id":2,
        "name":"Coffee-B",
        "price":1.99
    },
    {
        "id":3,
        "name":"Coffee-C",
        "price":3.25
    }
]
  • Listing a specific coffee by adding it's ID to the GET request path of rest/coffees (for example, rest/coffees/1]). This is how the output will look like: 
    {
    "id":1,
    "name":"Coffee-A",
    "price":2.75
}
  • Deleting a specific coffee by issuing a DELETE request to rest/coffees, adding the ID to the request path (for example, rest/coffees/1]).
  • Adding a coffee by issuing a POST request to rest/coffees. This is how the request body should look like (make sure to set the Content-Type header): 
    {
    "name":"Coffee-D",
    "price":3.33
}
